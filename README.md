# 12factorGo
12 factor app on go


1) Настройка Гитхаб и базы
2) Настроенное использование зависимостей(go mod)
3) Хранить данные в переменных окружениях (.env, Viper)
4) Одинаковый принцип конфигурации ко всем сервисам(базы данных - локлально или облако)
5) Релизы(heroku)
6) Процесс всегда stateless не сохраняем в памяти (stateful - Db, elastic, amqp)
7) Распределение портов
8) Декомпозиция сервиса
9) Ловить сигналы ОС и аккуратно остановить веб-сервис
